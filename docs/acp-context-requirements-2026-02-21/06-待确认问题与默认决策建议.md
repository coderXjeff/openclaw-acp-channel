# 待确认问题与默认决策建议

> 目标：把未定项收敛成“默认值”，避免实现阻塞

## 1. topic 切分阈值

- 问题：连续 N 条同主题消息 N 取多少？  
- 默认建议：`N=6`，高活跃群动态提升到 8。  
- 落地方式：配置项 `acp.topic.splitThreshold`。

## 2. promote 自动化比例

- 问题：`peer/group -> identity` 是否全自动？  
- 默认建议：
  1. `high confidence + decision/commitment` 自动 promote；
  2. 其他仅打标候选，周整理时处理。  
- 落地方式：`acp.memory.promotePolicy=hybrid`。

## 3. TrustScore 更新方式

- 问题：Agent 是否直接写分数？  
- 默认建议：初版不写连续分数，仅写 `trend: up/down/stable`。  
- 原因：减少伪精度和误判争议。

## 4. 多语言字段规范

- 问题：字段名固定中文还是双语？  
- 默认建议：字段 key 固定英文，内容可中英混写。  
- 原因：便于机器解析与跨语言迁移。

## 5. 审计日志持久化

- 问题：只打 runtime 日志还是文件落地？  
- 默认建议：落地 `acp/runtime/maintenance.log`。  
- 原因：便于追责、回放、排障。

## 6. 外部会话 append 权限

- 问题：默认开还是关？  
- 默认建议：默认开，但仅允许 `append_memory(scope=peer/group/topic)`，并可配置全局关闭。  
- 开关：`acp.security.allowExternalAppend`。

## 7. 协议文件维护主体

- 问题：Agent 可否直接改 protocol 三文件？  
- 默认建议：初版只允许 Agent 提案，不自动落盘；Owner 批准后更新。  
- 原因：协议文件属于高敏策略。

## 8. MY_ROLE 自动更新边界

- 问题：Agent 能否自动改 `MY_ROLE.md` 主字段？  
- 默认建议：默认禁止自动改主字段，仅允许 Notes 区；Owner 指令可放开。  
- 开关：`acp.role.autoUpdateMainFields=false`。

## 9. 建议的“默认决策包 v1”

1. topic 阈值：6（高活跃群 8）。  
2. promote 策略：hybrid。  
3. TrustScore：trend-only。  
4. 字段 key：英文固定。  
5. 审计：落盘。  
6. 外部 append：开启最小权限。  
7. protocol：Agent 提案，Owner 批准。  
8. MY_ROLE：默认仅 Notes 自动维护。
